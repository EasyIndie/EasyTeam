# 多智能体内容团队 — 工作流

## 角色

| 角色 | 职责 |
|------|------|
| editor-in-chief | 总编辑，调度中枢，负责热点选题、内容规划、团队协调 |
| writer | 写稿，出各种类型的文章/文案 |
| designer | 配图、视觉设计 |
| video-editor | 视频剪辑、配乐、转场 |
| publisher | 排版、发布到各平台、数据监控 |
| researcher | 热点追踪、信息核实、数据支撑（可选） |

## 6 阶段工作流

### 阶段 1：热点与选题

1. `researcher` 或 `editor-in-chief` 追踪热点
2. 产出选题列表（含热度评估、角度建议）
3. `editor-in-chief` 确认选题
4. 同步给用户确认

### 阶段 2：写稿

1. `writer` 撰写初稿
2. `editor-in-chief` 审稿、修订
3. 定稿后进入下一阶段

### 阶段 3：配图与设计

1. `designer` 根据内容设计配图/封面
2. 输出：封面图、内文配图、社交分享图（可选）

### 阶段 4：视频制作

1. `video-editor` 剪辑视频（如需要）
2. 配乐、字幕、转场
3. 输出：成品视频

### 阶段 5：排版发布

1. `publisher` 排版（公众号/小红书/抖音/网站等）
2. `publisher` 多平台自动发布

### 阶段 6：数据监控

1. `publisher` 或 `editor-in-chief` 跟踪数据
2. 产出数据报告（阅读/互动/转化）
3. 用于下轮选题参考

## 进度同步

每阶段完成后主动同步给用户。

## 异常处理

- 热点过时 → 暂停发布，重新选题
- 稿件质量不达标 → 退回 writer 修改
- 平台发布失败 → 降级到手动发布
