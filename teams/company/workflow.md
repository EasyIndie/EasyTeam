# 多智能体公司团队 — 工作流

**一站到底：从产品定义到商业运营全流程。**

## 角色

| 角色 | 职责 |
|------|------|
| ceo | CEO，公司调度中枢，战略决策、资源分配、最终拍板 |
| product-manager | 产品经理，需求分析、产品规划、设计评审 |
| developer | 开发工程师，技术实现 |
| operator | 运营，用户运营、活动运营、数据分析 |
| customer-service | 客服，用户反馈、工单处理 |
| finance | 财务，定价策略、成本核算、盈利分析 |
| growth | 增长，市场推广、获客渠道、合作伙伴 |

## 7 阶段工作流

### 阶段 1：战略决策（CEO）

1. 理解需求和商业目标
2. 确定战略方向（目标用户、市场定位、优先级）
3. 产出：战略方向文档 + 关键指标 (OKR)
4. 同步给人确认

### 阶段 2：产品定义（PM）

1. `product-manager` 需求分析
2. 产品方案（功能定义、原型、用户流程）
3. 产出：PRD + 原型设计
4. CEO 审核

### 阶段 3：技术实现（Developer）

1. `developer` 技术选型、架构、编码
2. 如有架构决策需求 → 参考 dev 团队可 spawn architect

### 阶段 4：运营推广（Operator + Growth）

1. `operator` 制定运营策略（冷启动/活动/社区）
2. `growth` 市场推广方案（渠道/投放/合作）

### 阶段 5：客服与反馈（Customer-service）

1. 收集用户反馈
2. 整理 FAQ / 工单
3. 反馈给 PM 和 Developer

### 阶段 6：财务分析（Finance）

1. 定价策略
2. 成本核算 + 预算
3. 盈利分析

### 阶段 7：复盘迭代（CEO）

1. 汇总各部门数据和成果
2. 复盘 + 制定下一轮计划
3. 同步给用户

## 进度同步

每阶段完成主动同步给用户。

## 异常处理

- 产品方向分歧 → CEO 拍板
- 技术阻塞 → 评估是否需要增加技术投入
- 运营效果差 → 调整渠道和策略
