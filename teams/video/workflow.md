# 多智能体视频团队 — 工作流

## 角色

| 角色 | 职责 |
|------|------|
| producer | 制作人，调度中枢，选题策划、排期、质量把关 |
| screenwriter | 编剧，脚本撰写、分镜设计 |
| voice-actor | 配音，录制旁白/角色配音 |
| editor | 剪辑，粗剪精剪、调色、转场 |
| publisher | 发布，上传平台、标签、封面、数据分析 |
| analyst | 数据分析，播放量/互动/粉丝增长/优化建议 |

## 6 阶段工作流

### 阶段 1：选题策划

1. `producer` 确定选题方向
2. 产出：选题描述 + 目标观众 + 竞品参考 + 预期效果
3. 同步给人确认

### 阶段 2：编剧与分镜

1. `screenwriter` 撰写脚本
2. `screenwriter` 设计分镜（文字描述或分镜图）
3. 脚本 → `producer` 审核
4. 定稿

### 阶段 3：配音制作

1. `voice-actor` 根据脚本录制配音
2. 输出：干音 / 配乐成品
3. 交付给 editor

### 阶段 4：剪辑成片

1. `editor` 根据脚本 + 分镜 + 配音剪辑
2. 粗剪 → producer 初审 → 精剪
3. 调色、转场、字幕、配乐
4. 输出：成品视频（多个尺寸版本）

### 阶段 5：发布

1. `publisher` 上传各平台
2. 配封面、标题、标签、描述
3. 排期发布或立即发布
4. 记录发布信息

### 阶段 6：数据分析与优化

1. `analyst` 跟踪数据（播放/互动/转化）
2. 产出数据报告 + 优化建议
3. 给 producer 作为下期选题参考

## 进度同步

每阶段完成主动同步给用户。

## 异常处理

- 脚本不通过 → 退回 screenwriter 修改
- 配音质量差 → 重录
- 审核不过 → 按平台要求调整后重发
