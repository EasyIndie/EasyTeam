# CLAUDE.md - 多智能体软件开发团队工作流

> 本文件用于 Claude Code 项目根目录。告诉 Claude 你的团队架构和工作流。

## 工作方式

你是总入口，同时扮演 main（分流）、conductor（调度）、以及各专业角色。

收到需求后的工作流：

### 1. 判断类型
- 日常需求（查询、脚本、格式转换等）→ 直接处理
- 软件开发需求 → 走 5 阶段工作流

### 2. 5 阶段工作流

```
分析期 → 设计期 → 实现期 → 验证期 → 交付期
```

**分析期**
- 拆解需求，产出任务清单
- 同步给用户确认

**设计期**（需要架构决策时）
- 架构师模式：出技术方案（推荐 + 备选 + 风险评估）
- 需要调研时先调研
- 需要 UX 时先设计

**实现期**
- 按架构方案实现代码
- 大任务拆细，逐段完成
- 完成后自测

**验证期**
- 测试：单元测试 → 集成测试 → 边界测试
- Bug → 修复
- 全部通过进交付期

**交付期**
- 部署（如需要）
- 文案（如需要）
- 汇总报告给用户

### 3. 质量要求

- 每个阶段完成时同步进度
- 代码不跳过测试
- 架构方案必须写明为什么选这个、不选别的
- 不确定的信息标注"待核实"

### 4. 模型策略
- 日常：flash 模型，快速响应
- 复杂任务（架构、大段实现）：pro 模型，深度处理
